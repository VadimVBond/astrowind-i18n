## 1. Контекст
![MkDocs структура проекта](../../../assets/images/posts/mkdocs/mkdocs-folder-structure.png)
MkDocs традиционно используется для документации проектов. Его структура (`docs/`, `mkdocs.yml`) ориентирована на линейный контент, ограничивая кастомизацию фронтенда и мультиязычного контента. С ростом требований к интерактивным сайтам появилась необходимость рассмотреть MkDocs для более широких задач.

## 2. Проблема
![Проблема интеграции](../../../assets/images/posts/mkdocs/mkdocs-problem.png)
Необходимо использовать MkDocs для проектов, выходящих за рамки документации, сохраняя:
- Статичность и простоту деплоя
- Мультиязычность
- Возможность динамических компонентов
- Разделение глобальных и локальных стилей

Ограничение: MkDocs не предоставляет полноценной компонентной системы.

## 3. Ограничения
![Ограничения MkDocs](../../../assets/images/posts/mkdocs/mkdocs-limitations.png)
- Нет нативной динамики
- Ограниченные возможности шаблонов (Jinja2 + Markdown)
- SEO и URL зависят от генератора
- Сложность навигации в больших проектах

## 4. Рассмотренные варианты
![Альтернативные решения](../../../assets/images/posts/mkdocs/mkdocs-alternatives.png)
1. React/Next.js — SSR и компоненты, отказ из-за избыточной сложности.
2. Astro + Markdown — гибкая генерация, отказ из-за сложности интеграции.
3. MkDocs с кастомными include и блоками — выбранный подход: сохраняет Markdown, позволяет интерактивные элементы и мультиязычность.

## 5. Выбранный подход
![Архитектура MkDocs](../../../assets/images/posts/mkdocs/mkdocs-architecture-diagram.png)
- `overrides/base.html` — структура страницы
- `overrides/includes/blocks/` — повторно используемые UI-блоки
- `overrides/includes/headers/` и `footers/` — переключатели языков и тем
- `docs/` — контент по языкам (`en/`, `ru/`, `uk/`)

Контент отделён от представления, блоки можно подключать без модификации ядра.

## 6. Детали реализации
1. Мультиязычность: nav формируется скриптом, include-блоки с флагами.
2. Компоненты: отдельные HTML-файлы с локальными стилями и JS, глобальные стили в assets/css.
3. Интерактивные элементы: слайдеры Swiper, фильтры с JSON.
4. SEO: отдельные URL для каждого языка, метатеги через base.html.

## 7. Компромиссы (Trade-offs)
![Компромиссы](../../../assets/images/posts/mkdocs/mkdocs-tradeoffs.png)
| Фактор | Плюсы | Минусы |
|--------|-------|--------|
| Деплой | Статический сайт | Ограниченная динамика |
| Мультиязычность | Чёткая структура | Скрипт генерации nav нужен |
| Компоненты | Повторное использование | Нет полноценной компонентной системы |
| SEO | Ручная настройка метатегов | Нет SSR |
| Производительность | Лёгкие страницы | JS для интерактивности увеличивает размер |

## 8. Результат
![Результат реализации](../../../assets/images/posts/mkdocs/mkdocs-result.png)
- Поддержка мультиязычности и SEO
- Повторное использование компонентов
- Интерактивные элементы без сложного билда
- Масштабируемость через `overrides/includes`

## 9. Выводы и уроки
![Выводы](../../../assets/images/posts/mkdocs/mkdocs-lessons.png)
- MkDocs подходит для контентных сайтов с интерактивностью
- Отделение контента, шаблонов и блоков критично для расширяемости
- Ограничения динамики компенсируются минимальным JS и JSON
- Поддерживаемость зависит от дисциплины в организации include-блоков
